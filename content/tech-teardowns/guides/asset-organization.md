# Asset Organization Guide

**Purpose**: Guide for organizing video, audio, image, and document assets in Tech Teardown projects.

---

## Overview

### Why Organize Assets?
- **Efficiency**: Find files quickly
- **Workflow**: Smooth production process
- **Backup**: Easy to backup important files
- **Collaboration**: Easy for others to understand
- **Scalability**: Works for multiple projects

### Organization Principles
- **Consistent**: Use consistent naming and structure
- **Logical**: Organize by type and purpose
- **Scalable**: Structure works for any project size
- **Clear**: Easy to understand and navigate

---

## Video Assets

### Raw Footage

#### Organization
- **Location**: `02-footage/raw/`
- **Structure**: Organize by scene/section
- **Naming**: `[section]-[description]-[take].mp4`

#### Sections
- **intro/**: Device introduction footage
- **disassembly/**: Disassembly process footage
- **components/**: Component examination footage
- **history/**: History narration footage
- **reassembly/**: Reassembly attempt footage

#### Naming Examples
- `intro-device-presentation-001.mp4`
- `disassembly-opening-case-001.mp4`
- `components-processor-closeup-001.mp4`
- `history-corporate-politics-001.mp4`
- `reassembly-attempt-001.mp4`

### Selected Footage

#### Organization
- **Location**: `02-footage/selects/`
- **Structure**: Organize by scene/section
- **Naming**: `[section]-[description]-select.mp4`

#### Purpose
- Best takes from raw footage
- Ready for editing
- Organized for easy access

### B-Roll Footage

#### Organization
- **Location**: `02-footage/b-roll/`
- **Structure**: Organize by type
- **Naming**: `[type]-[description].mp4`

#### Types
- **establishing/**: Establishing shots
- **cutaway/**: Cutaway shots
- **detail/**: Detail shots
- **atmosphere/**: Atmospheric shots

## File Naming Best Practices

### General Rules
- **Descriptive**: Names should describe content
- **Consistent**: Use consistent naming pattern
- **Sequential**: Use numbers for sequences (001, 002, etc.)
- **No Spaces**: Use hyphens or underscores
- **Lowercase**: Use lowercase (easier to work with)

### Naming Patterns

#### Video Files
- `[section]-[description]-[take].mp4`
- Example: `intro-device-presentation-001.mp4`

#### Audio Files
- `[type]-[section]-[version].wav`
- Example: `narration-intro-final.wav`

#### Image Files
- `[type]-[description]-[number].jpg`
- Example: `component-processor-001.jpg`

#### Document Files
- `[document-type].md`
- Example: `disassembly-log.md`

## Organization Workflow

### During Production
1. **Create Folders**: Set up folder structure at start
2. **Name Consistently**: Use naming conventions from start
3. **Organize Immediately**: Put files in correct folders right away
4. **Update Documentation**: Keep documentation current

### After Production
1. **Review Organization**: Check that everything is organized
2. **Clean Up**: Remove unnecessary files
3. **Archive**: Move to archive when complete
4. **Backup**: Backup important files

---

## Backup Strategy

### What to Backup
- **Raw Footage**: Original, unprocessed footage
- **Final Export**: Final video export
- **Project Files**: Editing project files
- **Documents**: All documentation
- **Graphics**: Final graphics assets

### Backup Locations
- **Local**: External hard drive
- **Cloud**: Cloud storage service
- **Archive**: Long-term archive location

### Backup Schedule
- **During Production**: Backup regularly (daily/weekly)
- **After Completion**: Full backup
- **Long-term**: Archive backup

2. **Stay Organized**: Keep files organized as you go
3. **Use Naming Conventions**: Consistent naming saves time
4. **Backup Regularly**: Don't lose work
5. **Archive When Done**: Keep projects organized long-term

## Common Mistakes to Avoid

### ❌ Poor Organization
- **Problem**: Files scattered, hard to find
- **Solution**: Use folder structure, organize immediately

### ❌ Inconsistent Naming
- **Problem**: Hard to find files, confusing
- **Solution**: Use naming conventions consistently

### ❌ No Backup
- **Problem**: Risk of losing work
- **Solution**: Backup regularly, multiple locations

### ❌ Not Archiving
- **Problem**: Projects pile up, hard to manage
- **Solution**: Archive completed projects
