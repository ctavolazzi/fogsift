# Feature: Testimonials & Social Proof Section

## Overview

Add a testimonials and social proof section to FogSift to build credibility and trust with potential clients. This is the #1 priority feature identified in the Feature Void Audit as the most impactful trust builder for consulting websites.

## Problem Statement

Currently, FogSift has no visible social proof. Visitors cannot:
- See who else has worked with FogSift
- Read about outcomes from previous engagements
- Verify credibility through third-party validation

**Impact:** Without testimonials, conversion rates are estimated at 50-70% lower than industry average.

## Target Audience

- C-suite executives, VPs of Operations, Directors of Strategy
- Mid-to-large companies ($10M-$500M+ revenue)
- Decision-makers evaluating consulting services

**What They Need to See:**
1. Real people who trusted this consultant
2. Measurable outcomes from engagements
3. Relevance to their industry/problem type

## Feature Requirements

### Must Have (MVP)
- [ ] Testimonials section between "About" and "Pricing"
- [ ] 3-5 testimonial cards with:
  - Quote text
  - Person's name
  - Title/Role
  - Company name (or anonymized industry)
  - Optional: Headshot/avatar
- [ ] Responsive layout (carousel on mobile, grid on desktop)
- [ ] Matches existing 70s retro-modern design system
- [ ] Accessible (ARIA labels, keyboard navigation)

### Should Have
- [ ] Client logos row (even anonymized/industry icons)
- [ ] "Results" callout (e.g., "15+ engagements completed")
- [ ] Subtle animation on scroll into view
- [ ] Link to full case studies in wiki

### Nice to Have
- [ ] Video testimonial embed option
- [ ] Rotating testimonials carousel
- [ ] Industry filter/tabs
- [ ] Schema.org Review markup for SEO

## Design Specifications

### Placement
Insert new section between "ABOUT" (Section 3) and "PRICING" (Section 4):

```
HERO → PROCESS → ABOUT → [NEW: TESTIMONIALS] → PRICING → FOOTER
```

### Visual Design

**Section Label:** "WHAT CLIENTS SAY" or "RESULTS"

**Card Design:**
```
┌─────────────────────────────────────────┐
│  "Quote text goes here. This should    │
│   be impactful and specific about      │
│   the outcome or experience."          │
│                                         │
│   ○ Name                               │
│   Title, Company/Industry              │
└─────────────────────────────────────────┘
```

**Color Palette (from tokens.css):**
- Background: var(--color-surface) or slight variant
- Card background: var(--color-surface-alt)
- Quote text: var(--color-text)
- Name: var(--color-accent) or var(--color-heading)
- Title: var(--color-text-muted)
- Quote marks: var(--color-primary) at 20% opacity

**Typography:**
- Quote: Inter, 1.1rem, italic
- Name: Outfit, 0.9rem, bold
- Title: Inter, 0.8rem, regular

### Responsive Behavior

**Desktop (>768px):**
- 3-column grid layout
- All testimonials visible

**Tablet (480-768px):**
- 2-column grid
- All testimonials visible

**Mobile (<480px):**
- Single column stack OR
- Horizontal scroll carousel with dots
